# Job to be Done 定義

## デザインガイドライン設計エージェント

### メインジョブ（Main Job）

**「競合調査結果をもとに、デザインの方向性と具体的なガイドラインを素早く策定し、クライアントに説得力のある提案を行いたい」**

---

## ジョブステートメント（詳細版）

| 項目 | 内容 |
|------|------|
| **When（状況）** | 競合調査・情報設計が完了し、デザイン制作に入る前 |
| **I want to（望むこと）** | 調査結果に基づいた論理的かつ一貫性のあるデザインガイドラインを作成したい |
| **So that（達成したい成果）** | デザインの意思決定に費やす時間を短縮し、クリエイティブな作業に集中できる |

---

## 機能的ジョブ（Functional Jobs）

1. **差別化の言語化**
   - 競合LPとの視覚的差別化ポイントを明確にしたい
   - サービスの特徴から「伝えるべき印象」をキーワード化したい

2. **デザイン方針の論理的根拠の確立**
   - なぜこのデザインなのか？をクライアントに説明できる根拠が欲しい
   - 感覚的ではなく、調査に基づいたコンセプトを持ちたい

3. **具体的なデザイン要素の決定**
   - 配色の組み合わせを決めたい
   - フォントタイプ・サイズの組み合わせを決めたい
   - イラスト・写真のトーン&マナーを決めたい

4. **参考事例の収集**
   - コンセプトに近い実際のデザイン事例を見つけたい
   - 過去の自社実績から類似案件を参照したい

---

## 感情的ジョブ（Emotional Jobs）

| ジョブ | 詳細 |
|--------|------|
| **自信を持って提案したい** | 論理的な根拠があることで、デザイン提案に説得力を持たせたい |
| **不安を減らしたい** | デザインの方向性が「なんとなく」ではなく、調査に裏付けられている安心感が欲しい |
| **クリエイティブに集中したい** | ガイドライン策定の作業から解放され、本来の創造的作業に時間を使いたい |

---

## 社会的ジョブ（Social Jobs）

| ジョブ | 詳細 |
|--------|------|
| **プロフェッショナルに見られたい** | クライアントに対して、体系的なアプローチで提案できる専門家として認識されたい |
| **チーム内での信頼を得たい** | 営業・ディレクターから「この人に任せれば安心」と思われたい |

---

## 期待する成果物（アウトプット）

以下の **3層構造** での出力が求められています：

```
┌─────────────────────────────────────────────────────────┐
│ 1. デザインで達成すべきゴール                            │
│    └─ キーワード例：信頼性、先進性、清潔感、堅実 など      │
├─────────────────────────────────────────────────────────┤
│ 2. デザインコンセプトの提言                              │
│    └─ ゴールを達成するための設計方針・世界観の言語化      │
├─────────────────────────────────────────────────────────┤
│ 3. デザインガイドライン                                  │
│    ├─ フォント種類（具体的なフォントファミリー）          │
│    ├─ 配色パレット                                       │
│    ├─ イラストのイメージ例                               │
│    └─ 写真のイメージ例                                   │
├─────────────────────────────────────────────────────────┤
│ 4. 参考実例                                              │
│    ├─ ギャラリーサイトからの類似デザインURL               │
│    └─ 自社実績からの類似案件URL                          │
└─────────────────────────────────────────────────────────┘
```

---

## ペインポイント（現状の課題）

1. **時間がかかる** - デザインガイドライン策定に多くの時間を費やしている
2. **属人化** - デザイナーの経験・センスに依存している
3. **根拠が弱い** - 競合調査と視覚デザインの接続が曖昧になりがち
4. **参考事例探しが手間** - 適切な参考事例を見つけるのに時間がかかる

---

## 成功指標（Success Metrics）

- デザインガイドライン策定時間の短縮
- クライアント提案時の説得力向上（受注率への寄与）
- デザイナーがクリエイティブ作業に集中できる時間の増加
- デザインの一貫性・調和の担保
